## Honest Feedback (100+ words, 3 friction points)
1. **Tmux Nudge Interference**: Sending `maw hey` nudges while an agent is actively working on a task interrupts the current conversation step in Codex. It behaves like an escape key or Ctrl-C, leading to confusion and partial tool execution. We must ensure agents are idle before nudging.
2. **CLI Mismatches**: Having some agents default to `codex` (which attempts self-updates and fails due to npm permission constraints) while others use `gemini` creates divergence in command compatibility, such as `/awaken` being unrecognized in Codex.
3. **Implicit Shell States**: Determining whether a tmux pane is running a bash shell or an active agent loop requires manual tracing of `maw peek` output, which can be verbose and hard to parse automatically.

## Lessons Learned
1. **Resuming Agent Loops**: When an agent exits to a raw bash shell, sending text to it just types commands. We must use `gemini` or `codex resume` to bring the agent loop back online before communicating.
2. **Nudges can Interrupt**: Do not send duplicate `maw hey` nudges while the target agent is showing 'Working...' state in the logs, as it causes conversation interrupts.
3. **Standardize on Gemini**: Newer Oracles should be run using `gemini` instead of `codex` to ensure modern slash command resolution (like `/awaken`) functions properly.
