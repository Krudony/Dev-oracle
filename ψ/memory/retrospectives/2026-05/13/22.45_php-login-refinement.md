## Honest Feedback
1. **Network Layer Ambiguity:** Debugging IP addresses through multiple layers (Cloudflare -> Docker Network -> PHP Server) is inherently noisy. I spent too much time trusting `docker logs` which showed the gateway IP, leading to a brief moment of doubt before I realized the application logic was actually working correctly. I need to trust application-level debug scripts more when dealing with complex headers.
2. **Reporting Friction:** The new reporting rules (talk-to + hey) added a bit of cognitive overhead mid-flow. It's a good practice for synchronization, but I need to automate or internalize the pattern so it doesn't break the "research-execution" cycle.
3. **Sqlite Utility Missing:** The lack of `sqlite3` inside the container was a minor annoyance. I had to resort to PHP one-liners to query the database, which is slightly slower than a direct shell query. I should probably add basic dev tools to my Dockerfiles in the future for easier debugging.

## Lessons Learned
- Always prioritize `HTTP_CF_CONNECTING_IP` when behind Cloudflare to avoid gateway IP capture.
- Timezone management in PHP should be explicit via `date_default_timezone_set` to avoid inconsistencies between app logic and DB defaults.
- Use `GROUP BY` and `HAVING MIN(created_at)` for robust first-time-seen detection in logs.
