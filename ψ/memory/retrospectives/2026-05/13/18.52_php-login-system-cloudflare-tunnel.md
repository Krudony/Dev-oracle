2. **Database Pathing:** The transition from running the PHP server in a specific directory to running it inside a Docker container broke the SQLite database path. I need to be more mindful of absolute vs. relative pathing when environment contexts change.
3. **Tunnel Configuration Sync:** The lag between updating the Cloudflare Dashboard and the `cloudflared` client picking up the new version (e.g., HTTP vs. HTTPS) caused unnecessary confusion and repetitive debugging.

## Lessons Learned
- **Container Networking in WSL2:** When exposing a local service to a Dockerized `cloudflared` instance in WSL2, using `--network host` for the tunnel container is often the most reliable way to bridge the network gap and reach exposed localhost ports from other containers.
- **Protocol Specificity in Tunnels:** Cloudflare Zero Trust demands strict alignment of protocols (HTTP vs. HTTPS). A mismatch will lead to instant 502/1033 errors, even if the port is correct.
- **Single Source of Truth:** Duplicate service installations (native vs. Docker) inevitably cause port conflicts or traffic misrouting. Regular environment audits are crucial when migrating deployment strategies.
